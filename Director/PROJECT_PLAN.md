# 📋 Детальный план проекта

## Цели проекта
1. Создать Telegram бота для приема заявок на автоматизацию
2. Предоставить AI-консультации по вопросам автоматизации
3. Минимизировать участие человека на этапе первичного контакта

## Функциональные требования

### F1: Прием заявок
- Бот собирает информацию через диалог
- Уточняющие вопросы зависят от предыдущих ответов
- Заявка сохраняется в БД
- Админ получает уведомление о новой заявке

### F2: AI Консультации
- AI ведет диалог как специалист по автоматизации
- Задает уточняющие вопросы
- Дает рекомендации
- Может предложить оставить заявку

### F3: Управление состояниями
- Пользователь может переключаться между режимами
- Контекст диалога сохраняется
- Команда /cancel отменяет текущий диалог

### F4: Администрирование
- Просмотр списка заявок
- Статистика
- Управление AI настройками

## Нефункциональные требования

### NF1: Производительность
- Время ответа бота < 2 секунд
- Поддержка 100+ одновременных пользователей
- Работа на 512MB RAM

### NF2: Надежность
- Автоматический рестарт при падении
- Логирование всех ошибок
- Бэкап БД

### NF3: Безопасность
- Валидация всех входных данных
- Защита от SQL инъекций
- Rate limiting

## Технические решения

### Почему aiogram 3.x?
- Полная асинхронность
- Встроенная FSM (Finite State Machine)
- Активная поддержка сообщества
- Хорошая документация

### Почему SQLite?
- Не требует отдельного процесса
- Минимальное потребление RAM
- Достаточно для < 100к заявок
- Простой бэкап (один файл)

### Почему не Docker (по умолчанию)?
- Docker добавляет overhead
- На маломощных серверах лучше напрямую
- systemd достаточно для управления

### AI Провайдер
- Основной: Google - Gemini-3.1-flash-lite-preview
- Альтернатива: Anthropic, OpenAI через LiteLLM
- Кэширование ответов для экономии

## Метрики успеха
- 80% заявок собираются полностью без человека
- Среднее время консультации < 5 минут
- Uptime > 99%
