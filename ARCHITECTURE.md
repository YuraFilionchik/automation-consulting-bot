## Поток данных

### Сценарий заявки:

```
User ──> /start
         │
         ▼
    Главное меню (inline кнопки)
         │
         └──> "📝 Оставить заявку"
              │
              ▼
         FSM: ApplicationState.project_type
              │
              ▼
         Выбор типа проекта (inline)
              │
              ▼
         FSM: ApplicationState.project_subtype (если бот)
              │
              ▼
         FSM: ApplicationState.budget_range
              │
              ▼
         FSM: ApplicationState.timeline
              │
              ▼
         FSM: ApplicationState.contact_info (текст)
              │
              ▼
         FSM: ApplicationState.task_description (текст)
              │
              ▼
         Сохранение в БД (application_service)
              │
              ├─> Показ резюме пользователю
              │
              └─> Уведомление админам (notification_service)
```

### Сценарий консультации:

```
User ──> /start
         │
         └──> "💬 Консультация с AI"
              │
              ▼
         FSM: ConsultationState.active
              │
              ▼
         User сообщение ──> ai_service.get_response()
              │                    │
              │                    ▼
              │              OpenAI API
              │                    │
              ▼                    ▼
         История диалога <── Ответ AI
         (в памяти)
              │
              ▼
         Показ ответа пользователю
              │
              ▼
         Цикл пока пользователь не уйдет
```

## Точки расширения

### Как добавить новый функционал:

1. **Новая команда**
   ```
   Создать handler -> Зарегистрировать в main.py
   ```

2. **Новый AI провайдер**
   ```
   Изменить ai_service.py -> Заменить OpenAI на другой API
   ```

3. **Новое состояние в заявке**
   ```
   ApplicationState (добавить State) -> application.py (добавить шаг)
   ```

4. **Новая БД**
   ```
   Заменить database.py -> Обновить модели
   ```

## Масштабирование

### Текущая архитектура (малый сервер):
- SQLite (один файл)
- MemoryStorage (в памяти)
- Polling (не webhook)

### Для роста нагрузки:

```
SQLite ──> PostgreSQL/MySQL
MemoryStorage ──> Redis
Polling ──> Webhook
Один процесс ──> Несколько воркеров
```
